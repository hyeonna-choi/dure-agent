A Dual-Agent Framework for Translating Unstructured Biological Experimental Protocols with Cross-Verification in Self-Driving Laboratories

Hyeonna Choi1, Jung Yup Kim2, Hyuneui Lim1, Seunggyu Jeon1
1Dept. of Bionic Machinery, AI Robot Research Institute, KIMM   2Nano-convergence Manufacturing Research Division, KIMM
  KSME Bio-Engineering Spring Conference 2026 · Poster
Poster Demo Video Code

A dual-agent framework translates a free-text protocol into executable equipment commands and runs it autonomously on the KIMM BioForge platform.

Abstract

Natural-language biological protocols are inherently unstructured — full of implicit procedural knowledge and ambiguous instructions — which makes them hard to execute directly on laboratory hardware. Converting them into machine-executable commands is a fundamental bottleneck for self-driving laboratories.

We propose a dual-agent framework: a Parser Agent structures the raw protocol, and a Validation Agent performs 3-stage cross-verification — completeness, parameter accuracy, and execution order — with iterative self-correction (up to 3 regenerations). The framework was deployed on the KIMM BioForge-1 platform, demonstrating fully automated bioassay execution directly from natural-language input.

Framework

  1. Parser Agent — converts an unstructured text protocol into a structured, step-by-step representation.
  2. Rule Engine — maps the structured output into concrete equipment-control command sequences.
  3. Validation Agent — cross-verifies the commands against the protocol on three axes (completeness, parameter accuracy, execution order) and emits PASS/FAIL with feedback.
  4. Auto-Regenerate (≤3×) — on FAIL, the framework self-corrects and regenerates, improving quality across iterations before execution on hardware.

Results

Protocol translation accuracy improves across validation iterations. Claude Sonnet 4.6 as validator yields the strongest iterative gains across 7 parser agents.

Rule-based (Standard) pipeline vs. direct LLM mapping across step completeness, parameter accuracy, and step-order accuracy.

End-to-end example: free-text protocol + well configuration → executed command sequence on BioForge-1.

System & Interface

Chatbot-style protocol input (PDF upload or direct text) with user-defined well allocation for standard-curve points, samples, and replicates.
